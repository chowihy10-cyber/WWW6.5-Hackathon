# VitalMint 观健


## 1. 项目概述

### 1.1 项目背景
女性健康管理长期存在 **数据孤岛**、**隐私顾虑** 与 **缺乏个性化干预** 三大痛点。现有 App 大多仅作记录，且将数据存储在中心化服务器上，用户无法真正掌控自己的健康数据，也难以获得可执行的、长周期的健康洞察。

### 1.2 核心理念
**“观照周期秩序，健造生命活力”**  
VitalMint观健 将女性生理周期及身体状态视为一个动态的、可解码的系统，通过 AI + Web3 技术，将健康行为与数据转化为可量化的生命活力资产。

### 1.3 产品定位
VitalMint观健 是一款 **专为女性设计的、基于 Web3 技术的个人健康智能管家**。  
- 目标用户：关注自身健康、饮食与作息质量的女性。  
- 差异化：隐私主权归用户（区块链）+ AI 驱动的周期预警与营养建议 + 通证激励闭环。

## 2. 核心功能模块

### 2.1 周期与症状追踪
- 记录月经周期、慢性疾病症状、情绪波动、皮肤状态（痤疮、湿疹等）。
- 支持自定义症状标签（如头痛、乳房胀痛、疲劳指数）。
- 日历视图 + 周期阶段预测（卵泡期、排卵期、黄体期、月经期）。

### 2.2 饮食与作息记录
- 拍照或手动录入饮食（AI 辅助识别营养素与炎症指数）。
- 睡眠时长 / 质量、压力水平、运动类型与时长，可选联动智能穿戴设备或手动录入。
- 与周期阶段联动，推荐「该阶段适合的抗炎食物或低强度运动」。

### 2.3 AI 健康洞察与预警
- **个性化基线建模**：基于至少一个完整周期的数据，建立用户的“正常波动范围”。
- **异常预警**：如周期推迟、异常出血、持续高炎症模式（基于 IL-6 / CRP 等代理指标推算）。
- **可执行建议**：例如“黄体期第 5 天，建议增加镁和 Omega-3 摄入，减少高升糖食物”。

### 2.4 Web3 隐私与数据主权
- 用户健康数据 **不上传中心化服务器**，而是加密后存储在去中心化存储网络（如 IPFS / Arweave），仅用户持有私钥。
- 可选择将 **匿名聚合数据** 贡献给科研机构（需用户签名授权），并获得通证奖励。
- 区块链记录 **数据哈希与访问日志**，确保可审计但不可篡改。

### 2.5 通证激励机制（VitalMint Token，VMT）
- **健康行为挖矿**：每日记录完整数据、完成建议动作（如抗炎饮食打卡、睡眠达标）获得 VMT。
- **数据贡献挖矿**：授权匿名数据用于女性健康研究，获得额外 VMT。
- **通证用途**：
  - 兑换合作品牌的女性健康产品（益生菌、月经杯、补剂）。
  - 参与 DAO 治理投票（决定科研资助方向或产品迭代优先级）。

## 3. 技术架构（计划）

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 客户端 | Flutter (iOS/Android) | 跨平台 + 本地加密数据库（SQLCipher） |
| AI 引擎 | TensorFlow Lite + 云端微调模型 | 本地推理周期预测 + 云上聚合模型更新（不接触原始数据） |
| 区块链 | Ethereum L2 (Polygon / Arbitrum) | 低成本存证 + 通证合约 |
| 存储 | IPFS + Filecoin | 用户加密数据分片存储，仅哈希上链 |
| 身份 | Web3Auth / WalletConnect | 低门槛钱包创建（支持邮箱社交恢复） |

---

## 4. 数据流程与隐私保护

1. **用户记录** → 本地加密 → 生成数据哈希  
2. **哈希上链**（用户签名） → 证明某时刻存在某组健康记录  
3. **加密数据** → 分片存储至 IPFS（用户控制解密密钥）  
4. **AI 推理**：  
   - 可选：下载公开聚合模型，本地运行推理（最隐私）   
5. **授权分享**：用户通过智能合约授权 → 科研机构获得临时解密权限 → 自动触发 VMT 奖励

## 6. 风险与应对

| 风险 | 影响 | 应对策略 |
|------|------|----------|
| 用户对 Web3 钱包操作陌生 | 高 | 采用社交恢复 + 免 Gas 交易（元交易） |
| AI 健康建议引发法律风险 | 高 | 明确声明非医疗设备，建议仅供参考；引入免责流程 |
| 通证价格波动影响激励预期 | 中 | 奖励锚定稳定币价值（如 1 VMT ≈ $0.01），回购机制 |
| 数据存储成本随用户增长上升 | 中 | 分阶段清理过期数据 + 用户付费扩容（VMT 支付） |
